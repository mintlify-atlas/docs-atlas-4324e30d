---
title: Apply metadata to a book
description: Write metadata fields directly onto a book record, replacing all provided fields unconditionally.
api: POST /api/books/{book_id}/apply-metadata
---

Applies the supplied metadata fields to the book, replacing every provided field regardless of whether the book already has a value. After updating the record, any configured rename and folder-organization rules are re-applied to the book file.

## Path parameters

<ParamField path="book_id" type="number" required>
  ID of the book.
</ParamField>

## Body

All body fields are optional. Only fields present in the request body are written; omitted fields are left unchanged.

<ParamField body="title" type="string">
  Book title.
</ParamField>

<ParamField body="author" type="string">
  Author name.
</ParamField>

<ParamField body="publisher" type="string">
  Publisher name.
</ParamField>

<ParamField body="published_date" type="string">
  Four-digit publication year (e.g. `"1965"`).
</ParamField>

<ParamField body="page_count" type="number">
  Total page count.
</ParamField>

<ParamField body="categories" type="string">
  Comma-separated subject categories.
</ParamField>

<ParamField body="language" type="string">
  Language code or name.
</ParamField>

<ParamField body="isbn" type="string">
  ISBN-10.
</ParamField>

<ParamField body="isbn13" type="string">
  ISBN-13.
</ParamField>

<ParamField body="rating" type="number">
  Numeric rating.
</ParamField>

<ParamField body="cover_url" type="string">
  URL of a cover image to download and save.
</ParamField>

<ParamField body="goodreads_id" type="string">
  GoodReads book ID.
</ParamField>

<ParamField body="series" type="string">
  Series name.
</ParamField>

<ParamField body="series_order" type="number">
  Position within the series.
</ParamField>

## Response

Returns the updated book object.

<ResponseField name="id" type="number" required>
  Book ID.
</ResponseField>

<ResponseField name="filename" type="string" required>
  Current filename (may change if rename rules are configured).
</ResponseField>

<ResponseField name="file_format" type="string">
  File extension, e.g. `epub`, `pdf`.
</ResponseField>

<ResponseField name="file_size" type="number">
  File size in bytes.
</ResponseField>

<ResponseField name="title" type="string">
  Book title.
</ResponseField>

<ResponseField name="author" type="string">
  Author name.
</ResponseField>

<ResponseField name="publisher" type="string">
  Publisher name.
</ResponseField>

<ResponseField name="published_date" type="string">
  Four-digit publication year.
</ResponseField>

<ResponseField name="page_count" type="number">
  Total page count.
</ResponseField>

<ResponseField name="language" type="string">
  Language code or name.
</ResponseField>

<ResponseField name="isbn" type="string">
  ISBN-10.
</ResponseField>

<ResponseField name="isbn13" type="string">
  ISBN-13.
</ResponseField>

<ResponseField name="rating" type="number">
  Numeric rating.
</ResponseField>

<ResponseField name="goodreads_id" type="string">
  GoodReads book ID.
</ResponseField>

<ResponseField name="cover_filename" type="string">
  Filename of the saved cover image, e.g. `42.jpg`. `null` if no cover.
</ResponseField>

<ResponseField name="series" type="string">
  Series name. `null` if not set.
</ResponseField>

<ResponseField name="series_order" type="number">
  Position within the series. `null` if not set.
</ResponseField>

<ResponseField name="date_added" type="string">
  ISO 8601 timestamp when the book was added.
</ResponseField>

<ResponseField name="date_modified" type="string">
  ISO 8601 timestamp of the last modification.
</ResponseField>

<ResponseField name="tags" type="string[]">
  Array of tag names currently applied to the book.
</ResponseField>

<RequestExample>

```bash cURL

  --url http://localhost:5000/api/books/42/apply-metadata \
  --header 'Content-Type: application/json' \

  "author": "Frank Herbert",
  "isbn13": "9780441013593",
  "published_date": "1965",
  "series": "Dune Chronicles",
  "series_order": 1
}'
```
