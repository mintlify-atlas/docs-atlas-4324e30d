---
title: Apply metadata to a book
description: Write metadata fields directly onto a book record, replacing all provided fields unconditionally.
---

`POST /api/books/{book_id}/apply-metadata`

Applies the supplied metadata to a book. Only `title`, `author`, `published_date`, `page_count`, and `cover_url` are written to the book record. If `cover_url` is provided, Bookie downloads and saves the image as the book's cover. After updating, any configured rename and folder-organization rules are re-applied to the book file.

<Note>
  This endpoint passes `replace_missing_only=False`, so the four text fields (`title`, `author`, `published_date`, `page_count`) are always overwritten with whatever value is in the request body, regardless of whether the book already has a value.
</Note>

## Path parameters

<ParamField path="book_id" type="integer" required>
  ID of the book.
</ParamField>

## Request body

All body fields are optional. Only fields present in the request body are processed; unrecognized fields are ignored.

<ParamField body="title" type="string">
  Book title. Replaces the current value unconditionally.
</ParamField>

<ParamField body="author" type="string">
  Author name. Replaces the current value unconditionally.
</ParamField>

<ParamField body="published_date" type="string">
  Publication year. Only the first 4 characters are stored (e.g. `"1965"`).
</ParamField>

<ParamField body="page_count" type="number">
  Total page count.
</ParamField>

<ParamField body="cover_url" type="string">
  HTTPS URL of a cover image to download and save. The image is downloaded via the scraper (HTTPS only, 10 MB limit) and saved as the book's cover JPEG.
</ParamField>

<Info>
  Other metadata fields such as `publisher`, `isbn`, `isbn13`, `language`, `description`, `rating`, `goodreads_id`, `series`, `series_order`, and `categories` are **not** updated by this endpoint. To update those fields, use `PUT /api/books/{book_id}` for the fields it supports (`title`, `author`, `published_date`, `page_count`, `series`, `series_order`, `rating`).
</Info>

## Response

Returns the updated Book object.

<ResponseField name="id" type="integer" required>
  Book ID.
</ResponseField>

<ResponseField name="filename" type="string" required>
  Current filename relative to the books directory (may change if rename rules are configured).
</ResponseField>

<ResponseField name="title" type="string">
  Updated book title.
</ResponseField>

<ResponseField name="author" type="string">
  Updated author name.
</ResponseField>

<ResponseField name="published_date" type="string">
  Four-digit publication year.
</ResponseField>

<ResponseField name="page_count" type="integer">
  Total page count.
</ResponseField>

<ResponseField name="cover_filename" type="string">
  Filename of the saved cover image (e.g. `42.jpg`), or `null` if no cover is set.
</ResponseField>

See [The Book object](/api-reference/overview#the-book-object) for the full response schema.

<RequestExample>

```bash cURL
curl --request POST \
  --url http://localhost:5000/api/books/42/apply-metadata \
  --cookie "session=<your-session-cookie>" \
  --header 'Content-Type: application/json' \
  --data '{
  "title": "Dune",
  "author": "Frank Herbert",
  "published_date": "1965",
  "page_count": 412,
  "cover_url": "https://covers.openlibrary.org/b/id/12345-L.jpg"
}'
```

</RequestExample>
